# PrivateVault Runtime Governance Demo

## Flow Demonstrated

1. AI evaluates risk (Grok)
2. PrivateVault intercepts the decision
3. Governance enforces runtime policy
4. Trace ID enables audit & replay

## Financial Enforcement Example

- Small transfer → ALLOW
- Unusual transfer → FLAG
- High-risk transfer → BLOCK

## Key Concepts

Runtime governance sits between AI decisions and execution.

AI decides.
PrivateVault controls what is allowed to execute.
